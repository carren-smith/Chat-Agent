## 5. All Methods by Category

### 5.1 Initialization Methods

#### `checkIsDesktop(): boolean`
- Checks `window.location.hostname` and `window.location.href`
- Returns `true` if hostname is `localhost` or `127.0.0.1`
- Returns `false` if hostname contains any known PowerBI service domain:
  `powerbi.com`, `app.powerbi.com`, `powerbi.cn`, `powerbi.us`, `powerbi.de`,
  `analysis.windows.net`, `powerbisandbox.com`, `pbidevs.com`
- On error: returns `true` (fail-open for desktop)

#### `initializeDOM(): void`
- Clears `this.target` children
- Creates DOM structure:

          #newChatBtn   (+ SVG icon, title="新建任务")

        textarea.chat-input ← this.inputElement (rows=1, placeholder="请输入您的问题...")

- Assigns `this.settingsButton`, `this.newChatButton`, `this.historyButton`,
  `this.messagesContainer`, `this.inputElement`, `this.sendButton`

### 5.2 PowerBI Lifecycle Methods

#### `update(t: VisualUpdateOptions): void`

Main PowerBI lifecycle hook. Called on every data/settings change.

**Execution sequence:**
1. Repopulates `formattingSettings` via service
2. Forces `aboutCardSettings.revertToDefault()` (branding lock)
3. Stores `this.dataView = t.dataViews[0]`
4. Reads `licenseSettings` from `dataView.metadata.objects`
5. Unmasks license key; if on Desktop, checks fingerprint match (clears key if mismatch)
6. If license key changed or invalid: calls `validateLicense(licenseKey, viewMode)`
7. Shows/hides `settingsButton` based on `viewMode === 1 && isDesktopEnv`
8. If only 1 assistant welcome message: updates welcome text from settings, re-renders
9. If `chatMessages` is empty: tries to restore session from history (by `lastActiveSessionId`),
   or loads most recent history entry, or calls `addWelcomeMessage()`

#### `getFormattingModel(): VisualFormattingModel`
- Returns `this.formattingSettingsService.buildFormattingModel(this.formattingSettings)`

### 5.3 License / Validation Methods

#### `getDeviceFingerprint(): string`
- Builds fingerprint from: `userAgent | language | platform | hardwareConcurrency | timezone`
- Computes a djb2-like hash over the joined string
- Returns `Math.abs(hash).toString(36)`
- On error: returns `"fallback_" + userAgent.length`

#### `async getOrCreateDeviceId(): Promise<string>`
- Tries `localStorage.getItem("abi_visual_fingerprint")`
- If not found: calls `getDeviceFingerprint()`, then `localStorage.setItem(...)` to cache
- Returns the fingerprint string

#### `async ensureLicenseValidated(): Promise<boolean>`
- If already valid: returns `true` immediately
- If no license key: calls `handleInvalidLicense(0)`, returns `false`
- If validation already in-flight: awaits existing `this.validationPromise`
- Otherwise: creates `this.validationPromise = performLicenseValidation(...)`, awaits it
- Returns `this.isLicenseValid`

#### `async validateLicense(licenseKey: string, viewMode: number): Promise<void>`
- Integrity check: verifies `performLicenseValidation.toString().length >= 50`
- If no key: calls `handleInvalidLicense(viewMode)`
- If key present and not already in progress: debounces 500ms then calls `performLicenseValidation`

#### `async performLicenseValidation(licenseKey: string, viewMode: number): Promise<void>`

Full validation flow:
1. Sets `isValidationInProgress = true`
2. Decodes `licenseKey` from base64, parses as JSON, checks `{name, exp}` fields
3. Calls license server:
   - **URL:** `https://abi-chat-verify-yxoranbhea.cn-hangzhou.fcapp.run` (hardcoded as base64)
   - **Method:** `POST`
   - **Body:** `{ licenseKey, deviceId, environment: "desktop"|"service" }`
4. On success (`code: 200, status: "valid"`):
   - Sets `this.activeSystemSecret = r.data.system_secret`
   - If `r.data.system_data` present: `this.systemPrompt = decryptPrompt(system_data, licenseKey)`
   - Clears `lastValidationError`
5. On failure: maps status codes to Chinese error messages, calls `handleInvalidLicense(viewMode, errorMsg)`
6. Always: sets `isValidationInProgress = false`, clears `validationDebounceTimer`

**License server response shape:**
```json
{
  "code": 200,
  "status": "valid" | "expired" | "limit_exceeded",
  "message": "string",
  "data": {
    "system_secret": "string",
    "system_data": "base64-encrypted-string"
  }
}
```

#### `handleInvalidLicense(viewMode: number, errorMsg?: string): void`
- Clears `activeSystemSecret` and `systemPrompt`
- Builds Chinese error message (with WeChat contact `powerai001`)
- Calls `showErrorMessage(msg)` — but only if the last message isn't already the same error

#### `decryptPrompt(encryptedData: string, key: string): string`
- Decodes `encryptedData` from base64 to bytes
- XOR-decrypts with UTF-8 bytes of `key` (repeating key)
- Returns decrypted UTF-8 string
- On error: returns `""`

#### `getExpertPrompt(): string`
- Returns `this.systemPrompt` if license valid and prompt set
- Fallback: `'你是"PowerBI星球打造的ABI Chat"。请注意，当前版本未激活或License无效，仅供演示使用。'`

#### `maskString(s: string): string`
- Returns `"ENC_" + btoa(encodeURIComponent(s))`
- Used before persisting API key and license key to PowerBI properties

#### `unmaskString(s: string): string`
- If starts with `"ENC_"`: returns `decodeURIComponent(atob(s.substring(4)))`
- Otherwise: returns `s` as-is

### 5.4 UI Rendering Methods

#### `adjustTextareaHeight(): void`
- Resets `inputElement` height to `"auto"`
- Sets height to `min(scrollHeight, 200)px`, minimum `36px`
- Sets `overflowY` to `"hidden"` or `"auto"` accordingly

#### `updateSendButtonState(): void`
- If `isGenerating`: changes button to red stop square SVG, title="停止生成", adds class `generating`
- Otherwise: restores send arrow SVG, title="发送", removes class `generating`

#### `addWelcomeMessage(): void`
- Creates welcome `ChatMessage` of type `"assistant"` with content from:
  1. `dataView.metadata.objects.welcomeSettings.welcomeMessage` (persisted)
  2. `formattingSettings.welcomeSettingsCard.welcomeMessage.value` (formatting panel)
  3. Hardcoded default: `"我是PowerBI星球打造的ABI Chat，帮助您从数据中挖掘价值，请告诉我您想了解什么内容？"`
- Pushes to `chatMessages`, calls `renderChatMessages()`, then `addSuggestionChips()`

#### `addSuggestionChips(): void`
- Reads 3 suggestion questions from:
  1. `dataView.metadata.objects.suggestionSettings.question1/2/3`
  2. `formattingSettings.suggestionSettingsCard.question1/2/3.value`
  3. Hardcoded defaults: `"分析目前上下文的数据"`, `"请设计一个分析框架"`, `"推荐适合的图表和可视化方案"`
- Filters out empty strings
- Creates `.suggestions-container` with `.suggestion-btn` buttons
- Clicking a button fills `inputElement.value` with the question text and adjusts height
- Appended after 50ms delay to the bottom of `messagesContainer`

#### `renderChatMessages(): void`
- Clears `messagesContainer`
- For each `ChatMessage` in `chatMessages`:
  - Creates `div.message.{type}` (type = `"user"` | `"assistant"` | `"loading"` | `"error"`)
  - For `"assistant"`: renders `formatToReportStyle(content)` as innerHTML + copy button
  - For others: sets `textContent` (safe)
  - If `chartData` present: creates `canvas#chart-{index}`, calls `renderChart()` after 100ms
- Scrolls `messagesContainer` to bottom

#### `sanitizeHTML(html: string): string`
- Uses `DOMParser` to parse HTML
- Whitelist of allowed tags: `h3 h4 p br div span table thead tbody tr th td ul ol li b strong i em u`
- Allowed attributes: `class style rowspan colspan width height align`
- Strips all other tags and attributes
- Falls back to `<` / `>` entity escaping on parse failure

#### `formatContentForDisplay(text: string): string`
- Strips leading ` ```html ` or ` ``` ` and trailing ` ``` ` fences
- Passes result through `sanitizeHTML()`

#### `formatToReportStyle(text: string): string`
- Checks if text contains a `````json` block matching:
  ```
  ```json
  { "intent": "data_query", ... }
  ```
  ```
- If found: extracts JSON, calls `executeDataQuery(queryObj)`, returns:
  `formatContentForDisplay(beforeBlock) + queryResult + formatContentForDisplay(afterBlock)`
- Otherwise: returns `formatContentForDisplay(text)`

#### `formatCellValue(val: any): string`
- `null`/`undefined` → `""`
- `number` → `val.toLocaleString()`
- Otherwise → `String(val)` with HTML entity escaping (`&`, `<`, `>`, `"`, `'`)

#### `showHistoryModal(): void`

Creates a full-screen modal (`div.tmdl-modal.history-modal`) with:
- Header: "历史任务" + close button (×)
- Body: list of history sessions from `ChatHistoryManager.loadHistory()`
- Each history item shows: icon, title, date, rename button, delete button
- **Rename**: replaces title span with inline input field (saves on Enter/blur)
- **Delete**: two-click confirmation (3s timeout to reset); calls `ChatHistoryManager.saveHistory()`
- Clicking a session item: calls `loadSession(session)` and removes modal
- Close button: removes modal from `document.body`

#### `showTmdlModal(): void`

Creates a full-screen modal (`div.tmdl-modal`) with 3 tabs:

**Tab 1 — License 配置:**
- Textarea for License Key (password-style masking via CSS, copy/cut disabled)
- Shows last validation error if present
- Includes anti-piracy: if Desktop+fingerprint mismatch, hides existing key

**Tab 2 — AI 连接配置:**
- Input: Base URL (reads from `aiSettings.apiUrl` or `formattingSettings`)
- Input: API Key (password type, reads and unmasks)
- Input: Model name

**Tab 3 — 数据模型上下文:**
- File upload button (accepts `.tmdl`, `.bim`, `.json`, `.txt`)
- Textarea editor (pre-filled from `TmdlManager.loadTmdl()`)

**Save button:** Persists all settings via `host.persistProperties()`:
```js
{
  merge: [
    { objectName: "aiSettings",     properties: { apiUrl, apiKey: masked, model } },
    { objectName: "licenseSettings", properties: { licenseKey: masked, boundFingerprint } }
  ]
}
```
Also calls `TmdlManager.saveTmdl(host, tmdlCode)`.
Validates license inline before switching to AI or TMDL tab.

**Tab switching guard:** Before switching away from License tab, calls inline license validation
(`performLicenseValidation` logic duplicated inline in modal).

### 5.5 Message Handling Methods

#### `async sendMessage(): Promise<void>`
1. Reads and trims `inputElement.value`
2. Returns early if empty
3. Calls `ensureLicenseValidated()` — returns early if invalid
4. Calls `addMessage("user", text)`
5. Clears input; calls `adjustTextareaHeight()`
6. Sets `isGenerating = true`, `updateSendButtonState()`
7. Creates `new AbortController()`
8. Calls `startStreamingMessage()`
9. Calls `callAIAPIWithStreaming(text, chunk => updateStreamingMessage(chunk))`
10. Calls `finalizeStreamingMessage(fullResponse)`
11. Calls `saveCurrentChatToHistory()`
12. On `AbortError`: saves history (user stopped)
13. On other error: removes streaming message, shows error
14. Finally: `isGenerating = false`, `updateSendButtonState()`, `abortController = null`

#### `addMessage(type: string, content: string, chartData?: object): void`
- Creates `{ type, content, timestamp: new Date(), chartData }`
- Pushes to `chatMessages`
- Calls `renderChatMessages()`

#### `showLoadingMessage(): void`
- Pushes `{ type: "loading", content: "正在思考中...", timestamp: new Date() }` to `chatMessages`
- Calls `renderChatMessages()`
- **Note:** Not used in the current streaming flow (legacy)

#### `hideLoadingMessage(): void`
- If last message has `type === "loading"`: pops it from `chatMessages`
- **Note:** Not used in the current streaming flow (legacy)

#### `showErrorMessage(text: string): void`
- If `isCapabilityBoundaryIssue(text)`: adds a canned "out of scope" assistant message
- Otherwise: pushes `{ type: "error", content: text }` to `chatMessages` and re-renders

#### `isCapabilityBoundaryIssue(text: string): boolean`
- Returns `true` if `text` contains any of: `不清楚、不确定、无法确定、不知道、超出、范围、复杂、具体、详细、个性化、定制、特殊需求`

#### `stopGeneration(): void`
- Calls `this.abortController.abort()`
- Sets `abortController = null`, `isGenerating = false`
- Calls `updateSendButtonState()`

### 5.6 Streaming Methods

#### `startStreamingMessage(): void`
- Creates empty `{ type: "assistant", content: "", timestamp: new Date() }` message
- Pushes to `chatMessages`
- Sets `currentStreamingMessageIndex = chatMessages.length - 1`
- Calls `renderChatMessages()`
- Gets reference to last `.message.assistant` DOM element → `streamingMessageElement`

#### `updateStreamingMessage(chunk: string): void`
- Appends `chunk` to `chatMessages[currentStreamingMessageIndex].content`
- Updates `streamingMessageElement` innerHTML via `formatToReportStyle()`
- While JSON block is open (` ```json ` found but no closing ` ``` `): shows "⏳ 正在思考并查询数据..." spinner
- Scrolls `messagesContainer` to bottom

#### `finalizeStreamingMessage(fullText: string): void`
- Sets `chatMessages[currentStreamingMessageIndex].content = fullText`
- Calls `parseAIResponse(fullText)` to extract chart data
- If chart data found: stores in `chatMessages[currentStreamingMessageIndex].chartData`
- Calls `renderChatMessages()`
- Resets `currentStreamingMessageIndex = -1`, `streamingMessageElement = null`

#### `removeStreamingMessage(): void`
- If `currentStreamingMessageIndex >= 0`: splices that message out of `chatMessages`
- Calls `renderChatMessages()`
- Resets streaming state

### 5.7 Data Processing Methods

#### `prepareDataContext(): string`
Builds the full text context sent to the AI with every user message:

1. Loads TMDL via `TmdlManager.loadTmdl()` — prepends if present
2. Returns `""` early if no `dataView.table`
3. Adds data overview block:
   ```
   数据概览：
   - 列数：N
   - 行数：N
   - 列名：col1, col2, ...
   ```
4. Computes pre-validated totals for numeric columns (non-percentage):
   ```
   【关键统计指标（已复核，请直接引用）】：
   - ColumnName: 总计=12,345
   ```
   Percentage columns flagged as `[比率指标] (不可直接汇总)`
5. Appends all rows line by line:
   ```
   1. col1: val1, col2: val2, ...
   2. ...
   ```

#### `executeDataQuery(queryObj: DataQuery): string`

Executes a structured query against `dataView.table`. Returns HTML table string.

**`DataQuery` object structure:**
```json
{
  "intent":       "data_query",
  "filters":      [{ "column": "ColName", "value": any, "operator": ">|<|>=|<=|==|!=|contains" }],
  "groupBy":      ["ColName1", "ColName2"],
  "aggregations": [{ "column": "ColName", "op": "sum|avg|count|max|min|first" }],
  "sort":         { "column": "ColName", "direction": "asc|desc" },
  "limit":        number
}
```

**Processing steps:**
1. Filter rows matching all `filters` conditions
2. If `groupBy` specified: group rows, compute aggregations per group
3. Sort by `sort.column` in specified direction (numeric or string comparison)
4. Limit to `limit` rows
5. Return HTML: `<div class="report-section">查询结果 (共 N 条)</div>` + `<table class="report-table">...</table>`

#### `parseAIResponse(text: string): { text: string, chartData?: ChartData }`
- Calls `formatToReportStyle(text)` for the formatted text
- If `shouldGenerateChart(text)`: calls `generateChartFromData(text)`, adds `chartData`
- Returns `{ text, chartData? }`

#### `shouldGenerateChart(text: string): boolean`
- Returns `true` if text contains any of 24 Chinese chart-creation phrases:
  `生成图表、创建图表、画图表、制作图表、显示图表、绘制图表、画个图表、做个图表、来个图表、要个图表、画柱状图、画折线图、画饼图、做柱状图、做折线图、做饼图、生成柱状图、生成折线图、生成饼图、创建柱状图、创建折线图、创建饼图、用图表显示、用图表展示、图表展示、图表呈现`

#### `generateChartFromData(text: string): ChartData | null`
- Requires `dataView.table` with at least 2 columns and 1 row
- Uses column[0] as labels, column[1] as data values
- Detects chart type from text keywords:
  - Contains `折线` or `趋势` → `"line"`
  - Contains `饼图` → `"pie"`
  - Contains `环形` → `"doughnut"`
  - Default → `"bar"`
- Returns Chart.js config object:
```json
{
  "type": "bar|line|pie|doughnut",
  "data": {
    "labels": ["col0 values..."],
    "datasets": [{
      "label": "col1 displayName",
      "data": [numeric values],
      "backgroundColor": ["#FF6384", "#36A2EB", "#FFCE56", "#4BC0C0", "#9966FF", "#FF9F40", "#FF6384", "#C9CBCF"],
      "borderColor": "#36A2EB",
      "borderWidth": 1
    }]
  },
  "options": {
    "responsive": true,
    "maintainAspectRatio": false,
    "plugins": { "legend": { "display": true } }
  }
}
```

#### `renderChart(canvas: HTMLCanvasElement, chartData: ChartData): void`
- Gets existing Chart instance from `this.charts.get(canvas.id)` and destroys it
- Creates new `Chart` (Chart.js `a.t1`) on the canvas 2D context
- Stores in `this.charts.set(canvas.id, chartInstance)`

### 5.8 API Call Methods

#### `async callAIAPI(userMessage: string): Promise<string>`
Non-streaming fallback (used internally for Gemini in streaming mode).

**Input resolution order for settings:**
1. `dataView.metadata.objects.aiSettings.{apiKey|apiUrl|model}` (persisted via settings modal)
2. `formattingSettings.aiSettingsCard.{apiKey|apiUrl|model}.value` (formatting panel)

**Provider detection:**
- If `apiUrl` contains `"googleapis.com"` OR model starts with `"gemini"` (case-insensitive) → Gemini mode
- Otherwise → OpenAI-compatible mode

**User message construction (when data available):**
```
数据上下文：
{prepareDataContext()}

用户问题：{userMessage}

请基于提供的数据回答用户问题。... [HTML format instructions]
```

**OpenAI request body:**
```json
{
  "model": "model-name (default: gpt-3.5-turbo)",
  "messages": [
    { "role": "system",    "content": "{systemPrompt}" },
    { "role": "user",      "content": "..." },
    { "role": "assistant", "content": "..." },
    "... (last 10 non-loading/error messages, excluding current) ...",
    { "role": "user",      "content": "{enriched user message}" }
  ],
  "temperature": 0.7
}
```

**Gemini request body:**
```json
{
  "contents": [{
    "parts": [{
      "text": "{systemPrompt}\n\n{historyAsText}\nUser: {enriched user message}"
    }]
  }],
  "generationConfig": { "temperature": 0.7 }
}
```
(No Bearer token for Gemini — API key is expected in the URL)

**OpenAI response parsing:** `response.choices[0].message.content`
**Gemini response parsing:** `response.candidates[0].content.parts[0].text`

#### `async callAIAPIWithStreaming(userMessage: string, onChunk: (chunk: string) => void): Promise<string>`
Streaming version (primary call path from `sendMessage()`).

- **For Gemini:** falls back to `callAIAPI()` then simulates streaming by iterating chars with 10ms delay
- **For OpenAI-compatible:** uses SSE streaming:
  - Request: same body as `callAIAPI` + `"stream": true`
  - Uses `AbortController` signal on the `fetch` call
  - Reads response body as `ReadableStream`
  - Parses `data: {...}` SSE lines
  - Extracts `choices[0].delta.content` from each chunk
  - Calls `onChunk(text)` for each non-empty chunk
  - On `AbortError`: returns accumulated text so far
  - Returns full accumulated response string

#### `getConversationHistoryMessages(): { role: string, content: string }[]`
- Filters `chatMessages` to exclude `"loading"` and `"error"` types
- Excludes the last message (current user message being sent)
- Takes the last 10 items
- Maps to `{ role: "user"|"assistant", content }`

#### `getPowerBIStarSystemPrompt(): string`
- Returns `this.systemPrompt` if `isLicenseValid && systemPrompt`
- Otherwise returns default demo prompt

### 5.10 Chat Session Methods

#### `generateSessionId(): string`
- Returns `Date.now().toString(36) + generateRandomString(8)`

#### `generateRandomString(length: number): string`
- Uses `window.crypto.getRandomValues(new Uint32Array(length))` if available
- Falls back to `Math.random().toString(36)`
- Returns `length`-character string

#### `async startNewChat(): Promise<void>`
1. If `chatMessages.length > 0`: calls `saveCurrentChatToHistory()`
2. Generates new `sessionId`
3. Sets `this.currentSessionId = sessionId`
4. Clears `chatMessages = []`
5. Calls `renderChatMessages()`
6. Calls `addWelcomeMessage()`
7. Calls `saveLastActiveSessionId(sessionId)`
8. `storageService.remove("currentSession")` if available

#### `async saveCurrentChatToHistory(): Promise<void>`
1. Returns early if no `"user"` type messages exist
2. Loads existing history via `ChatHistoryManager.loadHistory(dataView.metadata.objects)`
3. Creates/updates session entry:
   ```json
   {
     "id":          "sessionId",
     "title":       "First 15 chars of first user message + ...",
     "messages":    [...chatMessages],
     "lastUpdated": new Date()
   }
   ```
4. If session already in history: replaces it (moves to front); otherwise prepends
5. Calls `ChatHistoryManager.saveHistory(host, updatedHistory)`
6. Calls `saveLastActiveSessionId(currentSessionId)`

#### `loadSession(session: ChatSession): void`
- Sets `this.currentSessionId = session.id`
- Restores `chatMessages` from session (re-parses timestamps as `Date`)
- Calls `renderChatMessages()`
- Calls `saveLastActiveSessionId(session.id)`
- Calls `saveCurrentChatToHistory()`

### 5.11 Legacy Chat History Methods (localStorage-based)

These methods use multi-tier storage (PowerBI → localStorage → sessionStorage).
They appear to be legacy / unused in the primary flow (which uses `ChatHistoryManager` / `persistProperties`).

#### `async saveChatHistory(): Promise<void>`
- Serializes `chatMessages` to JSON
- Tries: `storageService.set("chatHistory", json)` (PowerBI storage)
- Falls back: `localStorage.setItem("powerbi-chat-history", json)`
- Falls back: `sessionStorage.setItem("powerbi-chat-history", json)`

#### `async loadChatHistory(): Promise<void>`
- Returns early if `chatMessages` already populated
- Tries loading from: PowerBI storageService → localStorage → sessionStorage
- Key used: `"chatHistory"` (PowerBI) and `"powerbi-chat-history"` (browser storage)
- Parses JSON, restores timestamps, calls `renderChatMessages()`

#### `async clearChatHistory(): Promise<void>`
- Removes from all three storage tiers
- Clears `chatMessages = []`, re-renders

### 5.12 Clipboard Methods

#### `async copyToClipboard(html: string): Promise<void>`
- Tries `navigator.clipboard.write([ClipboardItem({ "text/html": Blob, "text/plain": Blob })])` (secure context)
- Falls back to `fallbackCopyToClipboard(html)`

#### `fallbackCopyToClipboard(html: string): void`
- Creates off-screen `contentEditable` div with `innerHTML = html`
- Uses `document.execCommand("copy")` after `getSelection().addRange()`
- Cleans up the div

## 6. Supporting Static Classes

### 6.1 `TmdlManager` (module `267`, exported as `z`)

Manages TMDL (Tabular Model Definition Language) file storage in PowerBI properties.

| Constant | Value |
|----------|-------|
| `MAX_CHUNK_SIZE` | `25,000` chars per chunk |
| `MAX_CHUNKS` | `50` chunks max (~1.25MB total) |

#### `static cleanTmdl(tmdlCode: string): string`
- Strips `//` comments
- Strips `annotation` lines
- Strips metadata-only properties: `lineageTag`, `ordinal`, `sourceLineageTag`, `formatString`, `displayFolder`, `isAvailableInMdx`, `isPrivate`, `isHidden`, `changedProperty`, `dataCategory`, `summarizeBy`
- Collapses blank lines

#### `static saveTmdl(host: IVisualHost, tmdlCode: string): void`
1. Cleans the code via `cleanTmdl()`
2. Splits into 25KB chunks
3. Persists via `host.persistProperties({ merge: [{ objectName: "tmdlSettings", properties: { chunkCount, chunk0..chunkN }, selector: null }] })`

#### `static loadTmdl(objects: DataViewObjects): string`
- Reads `objects.tmdlSettings.chunkCount`
- Reassembles chunks `chunk0` through `chunk{N-1}`
- Returns full TMDL string (or legacy `tmdlCode` field if `chunkCount === 0`)

---

### 6.2 `ChatHistoryManager` (module `332`, exported as `S`)

Manages chat history persistence in PowerBI properties (chunked, up to ~1.25MB).

| Constant | Value |
|----------|-------|
| `MAX_CHUNK_SIZE` | `25,000` chars per chunk |
| `MAX_CHUNKS` | `50` chunks max |

#### `static saveHistory(host: IVisualHost, sessions: ChatSession[]): void`
1. Serializes sessions array to JSON
2. Chunks into 25KB pieces
3. Persists via `host.persistProperties`:
   ```js
   objectName: "historySettings",
   properties: { chunkCount: N, chunk0: "...", chunk1: "...", ..., chunk49: "" }
   ```
4. Throws `"History storage full"` if more than 50 chunks needed

#### `static loadHistory(objects: DataViewObjects): ChatSession[]`
- Reads `objects.historySettings.chunkCount` and reassembles chunks
- Parses JSON, re-hydrates `lastUpdated` and all message `timestamp` fields as `Date` objects
- Returns `[]` on error or missing data

## 11. External API Calls

### License Validation Server
- **URL:** `https://abi-chat-verify-yxoranbhea.cn-hangzhou.fcapp.run`
- **Method:** `POST`
- **Headers:** `Content-Type: application/json`
- **Request:**
  ```json
  {
    "licenseKey":   "base64-encoded-JSON",
    "deviceId":     "fingerprint-string",
    "environment":  "desktop" | "service"
  }
  ```
- **Response:**
  ```json
  {
    "code":    200,
    "status":  "valid" | "expired" | "limit_exceeded",
    "message": "...",
    "data": {
      "system_secret": "string",
      "system_data":   "XOR-encrypted-base64-prompt"
    }
  }
  ```
- Called from: `performLicenseValidation()`, `showTmdlModal()` (inline duplicate)

### AI API (OpenAI-compatible)
- **URL:** User-configured `apiUrl`
- **Method:** `POST`
- **Headers:** `Content-Type: application/json`, `Authorization: Bearer {apiKey}`
- **Non-streaming body:**
  ```json
  {
    "model":       "user-configured (default: gpt-3.5-turbo)",
    "messages":    [
      { "role": "system",    "content": "systemPrompt" },
      { "role": "user",      "content": "..." },
      { "role": "assistant", "content": "..." },
      "...last 10 messages...",
      { "role": "user",      "content": "enriched message with data context" }
    ],
    "temperature": 0.7
  }
  ```
- **Streaming body:** same + `"stream": true`

### AI API (Gemini)
- **Detected by:** `apiUrl` contains `"googleapis.com"` OR model name starts with `"gemini"`
- **No `Authorization` header** (API key expected in URL)
- **Request:**
  ```json
  {
    "contents": [{
      "parts": [{
        "text": "{systemPrompt}\n\n{historyText}\nUser: {enrichedMessage}"
      }]
    }],
    "generationConfig": { "temperature": 0.7 }
  }
  ```
- **Note:** Streaming for Gemini is **simulated** — calls `callAIAPI()` and replays characters with 10ms delay
